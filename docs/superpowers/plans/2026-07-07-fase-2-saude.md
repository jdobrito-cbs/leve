# FASE 2 — Integração com saúde — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Importação opt-in de peso e passos do Health Connect (Android) e HealthKit (iOS) atrás da interface `HealthProvider` existente, com sync manual, dedup e card de passos no Hoje.

**Architecture:** Providers nativos como adapters finos (mockados em teste); factory por plataforma com fallback `UnavailableHealthProvider` quando o módulo nativo não existe (Expo Go segue funcionando). Serviço `healthSync` concentra mapeamento/dedup (testável com better-sqlite3). Conexão registrada em `settings['health']`.

**Tech Stack:** react-native-health-connect + expo-health-connect + expo-build-properties (Android); @kingstinct/react-native-healthkit + react-native-nitro-modules (iOS).

## Global Constraints

- Mesmas regras das fases anteriores (strings em pt-BR.ts, commit+push por task, TDD, Testing Library v14, mocks com prefixo `mock`).
- Nenhuma permissão pedida sem ação do usuário; textos neutros; dados de saúde nunca para publicidade.
- `getHealthProvider()` NUNCA lança por módulo ausente — sempre retorna provider utilizável (fallback Unavailable).

**Interfaces:** Produces `getHealthProvider(): HealthProvider` — Android→HealthConnectProvider, iOS→HealthKitProvider, senão/fallha→Unavailable. Providers implementam a interface existente (`WeightSample { kg, takenAt, source }`, `StepsSample { count, date }`).

- [ ] Implementar providers (adapters):
  - HealthConnect: `initialize()`; `requestPermission([{accessType:'read',recordType:'Weight'},{...Steps}])`; `readRecords('Weight',{timeRangeFilter:{operator:'between',startTime,endTime}})` → `WeightSample{ kg: r.weight.inKilograms, takenAt: new Date(r.time), source:'healthconnect' }`; Steps somados por dia local.
